Order Service

Introduction

This is a Spring Boot application that demonstrates how to build a RESTful service with CRUD operations, using the power of Spring Security and OAuth server Keycloak for authentication and authorization, and using PostgreSQL for data persistence.

The service manages CRUD operations for Order and Product, and exposes the following CRUD APIs:

  • get products: to get products list.
  • create order: to create new order.
  • update order: to update existing order (update the PENDING order only).
  • delete order: to delete existing order (delete the PENDING order only).
  • get orders: to get user orders list (depending on the user email from the token).
  • get order by ID: to get order by ID.

Features

Security & OAuth

Using Spring Security and Keycloak OAuth server for authentication and authorization.

CRUD Operations

CRUD operations using Spring Boot for managing orders and products.

Database

PostgreSQL as the database with Liquibase for migrations.

Messaging

Integration with RabbitMQ for messaging.

Caching

Using Spring Cache and Hazelcast for caching.

API Documentation

Using OpenAPI for API documentation.

Database Auditing

Enabling database auditing for better traceability.

Code Generation

Using Lombok for code generation and Mapstruct for object mapping.

Containerization

Using Docker for containerization of the application.

Prerequisites

Before running the application, ensure you have the following installed:

  • Java 17+ (Recommended)
  • Maven 3.8+
  • PostgreSQL (either locally or via Docker)
  • Docker (optional, for PostgreSQL container)

Setup Instructions

Follow these steps to get the service running:

git clone https://github.com/ahmed-baz/order-service.git
cd order-service/docker
docker compose up -d

Login to Keycloak at http://localhost:9696 with admin/admin.

  • Create a realm order-service
  • Create a client order-api
  • Create a new user to test the service
